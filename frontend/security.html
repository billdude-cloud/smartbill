Security at SmartBill

Your data security is my top priority. Learn how I protect your information.

Last Updated: March 2024

Security Overview

SmartBill is built with security at its core. I implement multiple layers of protection to ensure your personal information and financial data remain safe and private.

HTTPS Encryption
Password Hashing (bcrypt)
JWT Authentication
Regular Security Audits

Encryption

All data transmitted between your device and our servers is encrypted using HTTPS/TLS 1.3.

Password Security

Passwords are hashed using bcrypt with salt rounds before being stored.

JWT Tokens

Secure JSON Web Tokens for authentication with 7-day expiration.

Database Security

Parameterized queries prevent SQL injection attacks.

Secure Storage

User data stored in MySQL with proper access controls.

Monitoring

Continuous monitoring for suspicious activities.

Authentication & Authorization

How Authentication Works

When you log in to SmartBill:

  • Your credentials are sent over HTTPS
  • Password is verified against the stored hash
  • A JWT token is generated and sent to your browser
  • The token is stored securely and sent with each request
  • Tokens expire after 7 days for security

Note: Your password is never stored in plain text. Even I cannot see your password.

Authorization Levels

  • Regular Users: Can create/manage their own bills and groups
  • Group Admins: Additional privileges to manage group members

Data Protection

What Data is Protected

  • Personal information (name, email)
  • Bill details and payment records
  • Group memberships
  • Profile pictures

How Data is Protected

  • Encrypted in transit (HTTPS)
  • Encrypted at rest in database
  • Regular automated backups
  • Firewall protection

GDPR Compliant: We respect your privacy rights and provide tools to access, correct, or delete your data.

Password Security

Password Requirements

  • Minimum 6 characters
  • Encouraged to use mix of letters, numbers, and symbols
  • Passwords are hashed with bcrypt
  • Password reset via secure email tokens

Password Reset Security

  • Reset links sent via email
  • Links expire after 1 hour
  • One-time use tokens
  • Cannot be reused

Avatar Upload Security

  • File type validation (only images allowed)
  • File size limit (5MB maximum)
  • Images are resized and optimized
  • Old avatars are automatically deleted

Session Management

  • Sessions expire after 7 days
  • Tokens are validated on each request
  • Logout invalidates the token
  • No session data stored on server

Protection Against Common Vulnerabilities

SQL Injection Prevention

All database queries use parameterized statements, making SQL injection impossible.

XSS Protection

User input is sanitized and escaped before rendering.

CSRF Protection

JWT tokens provide built-in CSRF protection.

Rate Limiting

API endpoints have rate limiting to prevent brute force attacks.

Security Best Practices for Users

What You Can Do

  • Use a strong, unique password
  • Never share your password with anyone
  • Log out when using shared devices
  • Keep your email account secure
  • Report suspicious activity immediately

Warning: I will never ask for your password via email or message. Be cautious of phishing attempts.

Incident Response

In case of a security incident:

  • Affected users will be notified within 72 hours
  • Affected accounts may be temporarily suspended
  • Password resets will be forced if necessary
  • Full investigation will be conducted

Reporting Security Vulnerabilities

If you discover a security vulnerability in SmartBill, please report it responsibly:

  • Email me directly at billodhiambo483@gmail.com
  • Provide detailed steps to reproduce the issue
  • Allow reasonable time for fixes before disclosure
  • I appreciate responsible disclosure and will acknowledge your help

Note: I take security seriously and appreciate the security community's help in keeping SmartBill safe.
